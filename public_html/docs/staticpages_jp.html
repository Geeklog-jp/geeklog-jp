静的ページプラグイン

概要

静的ページプラグインは，本来，（Geeklogが作成する動的ページとは対照的に）静的コンテンツを持ったページを作成するためのものであった。そういうわけで「静的」という名前になっている。「このサイトについて」や「目標宣言」などのページが思い浮かぶであろう。

その時以来，人々は様々な目的で静的ページを使用している。静的ページにPHPのコードを含めることができるようになっているので，静的ページという名前はもはやだいぶ不適切なものになっている。

機能

  • PHPの使用をサポート
  • 読みやすいURLを作成するための編集可能なページID
  • 静的ページをGeeklogのインデックスページと話題ページに表示し，全画面表示することもできる("スプラッシュスクリーン機能")
  • 既存のページをコピーする機能
  • GeeklogのURL書き換え
  • [staticpage:] と [staticpage_content:] 自動タグを実装
  • Geeklogのパーミッションシステムを適切にサポート

PHPの使用

PHPを使えるようにする

重要: セキュリティ上の理由で，静的ページでのPHP使用はデフォルトでは無効になっている。有効にするには，Static Page Adminグループに， staticpages.PHP 権限を与えなければならない。そのためには，管理者としてログインし，管理者用メニューから「グループ」を選ぶ。Static Page Adminを見つけて，グループ名の左側にある編集アイコン（ペンの形）をクリック。ページの下の方に「権限(Rights)」のリストがあり，staticpages.delete と staticpages.edit には既にチェックが入っている。Static Page Adminグループに属する人が静的ページでPHPを使えるようにするには， staticpages.PHP にチェックを入れてから保存をクリックする。

上述の staticpages.PHP 権限に加えて，静的ページ全体でPHPの使用を無効にするグローバルなオプションもある。このオプションは「管理者用メニュー - コンフィギュレーション - 静的ページ」の中にある「PHPを許可する」である。「いいえ」にすると，staticpages.PHP よりも優先され，静的ページの中でPHPを使えなくなる。

使用法

静的ページでPHPを不適切に使用すると，セキュリティの問題が発生する可能性がある。注意して使用すること。

静的ページごとにPHPの使用を有効にする必要がある。コンテンツ編集欄の下に，次の選択肢のあるPHPのドロップダウンリストがある。

  • PHPを実行しない
    静的ページの中にPHPのコードが含まれていても，実行されず，コードがそのまま表示される。
  • PHPを実行する(return)
    静的ページに含まれるPHPのコードが実行される。'return'とは，このページが表示する内容をPHPのコードで（echo 文や print 文を使って）直接出力する代わりに return 文で返すことを意味する。これは静的ページプラグイン ver 1.3から導入されたオプションである。
  • PHPを実行する
    静的ページに含まれるPHPのコードが実行される。このオプションを使うと，ページレイアウトに干渉することなくPHPの echo 文や print 文を使ってページの内容を出力できる。

静的ページにPHPのコードを埋め込むときに，PHPの <?php タグと ?> タグを記述してはいけない。静的ページに含まれるPHPのコードは，Geeklog側でこれらのタグを付加されて実行されるからである。

上述の「PHPを実行する」（returnなしの方）を選択する場合は，コードの中でPHPとHTMLを次のように行き来することができる。

echo "Hello"; ?>, <b>world</b>, <?php echo "how are you?";

この例では， "Hello, world, how are you?" と表示される。

ページID

静的ページを新規作成すると，ページIDが自動的に割り当てられる。このIDは日付・時刻と最大3桁のランダムな数字で構成されている。このページのURLを頻繁に使用するつもりなら（Eメールで引用する場合など），もっと読みやすいページIDにするのは意味のあることであろう。

「静的ページの編集」では，ページIDを変えることもできる。たとえば，「当サイトに関して」のページのIDを "20030313100131123" から "about" に変えて，次のURLにできる。

http://yoursite/staticpages/index.php?page=about

こちらの方が遙かに引用しやすいし，覚えやすい。GeeklogのURLリライト機能を活用してさらに改善することもできるだろう。

ページIDで安全に使える文字はアルファベットと数字だけである。各国固有の文字（ひらがな，カタカナ，漢字など）やほとんどの句読点(とはいえ，'-' と '.' は使えるはず)，スペースは使用しないこと。「静的ページの編集」は一部の不正な文字を検出するが，全部というわけではないので。

静的ページをインデックスページで使用する

Geeklogのセンターブロックの概念では，プラグインがサイトのセンターエリア（つまり，記事と記事の間）にブロックを出力することを許可している。

「静的ページの編集」で「センターエリア」にチェックを入れると，静的ページを表示する場所を選択するために，「表示エリア」と「話題」というドロップダウンリストが使えるようになる。「表示エリア」で選べるのは，「ページの最上部」，「注目記事の下」，「ページの下」（この3つは説明不要であろう）と「ページ全体」である。最後の「ページ全体」とは，この静的ページがインデックスページとして表示される，つまり，通常の記事が一切表示されなくなるということである。サイトの「スプラッシュ画面」や「ようこそ」ページを作成するのに役に立つだろう。

ヒント: 静的ページをスプラッシュ画面として使用する場合，訪問者を普通のインデックスページへ誘導するリンクが必要になることもあるだろう。この場合は，静的ページの中に index.php?display=all というリンクを作成すればよい。

2番目の「話題」ドロップダウンリストを使うと，静的ページの表示を特定の話題，ホームページ（トップページ）のみ，全ページ（つまり，すべての話題 + ホームページ）に限定できる。これはブロックを表示するオプションと全く同じである。

ヒント: これらのオプションをパーミッションと組み合わせることもできる。たとえば，ゲストユーザにしか表示されない「ようこそ」ページを作成することも可能である。

ソート（並べ換え）

センターブロックでのソート項目: センターエリアの同一セクション内（たとえば，インデックスページのトップに静的ページを2つ表示する場合）に複数の静的ページを表示させる場合，「管理者用メニュー - コンフィギュレーション - 静的ページ」で表示順を選ぶことができる。選択肢は，次の通り。

  • 日付（更新日時の新しい順に並べ換え）
  • ページID（ページIDで並べ換え：初期値）
  • タイトル（ページのタイトルで並べ換え）

この順序が適用されるのは，センターブロック内の静的ページだけである。センターブロックを出力する他のプラグインはこの順序には含まれない。また，センターブロックはふつうのGeeklogのブロックのような外見をしているとは限らない。この目的のためには，下にある「ページをブロックで囲む」を「はい」にすること。

メニューでのソート項目: サイトのメニューに表示される静的ページを並べ換えることもできる(header.thtml テンプレートファイル内で， {plg_menu_elements} 変数を使用している場合)。この項目も「管理者用メニュー - コンフィギュレーション - 静的ページ」で表示順を選ぶことができる。選択肢は，次の通り。

  • 日付（更新日時の新しい順に並べ換え）
  • ラベル（メニューに表示される文字列：初期値）
  • ページID（ページIDで並べ換え：初期値）
  • タイトル（ページのタイトルで並べ換え）

静的ページをブロックで囲む

「静的ページの編集」で「ブロックで囲む」をチェックすると，静的ページをGeeklogのブロックレイアウトで囲むことができる。この場合，静的ページのタイトルがブロックのタイトルとして使用される。また，普通のブロックと同様にヘルプ表示用のURLを入力することもできる。URLを入力すると，そのURLにリンクするヘルプアイコンが表示される。

このオプションの初期値は，「管理者用メニュー - コンフィギュレーション - 静的ページ」で設定することができる。

静的ページをコピーする

同じような静的ページがたくさんある場合，既存のページをコピーしてから編集したいだろう。静的ページのリストで，コピー欄のアイコンをクリックすれば簡単にコピーできる。コピーすると新たなページIDがついたページができる。

URLリライト

この機能は実験的なものであり，IISでは動作しないことが知られている。

Geeklogはある種のURL書き換え，つまり，URLの外観を検索エンジンに拾われやすいものにする機能を持っている。たとえば，

http://yoursite/staticpages/index.php?page=20030313100131123

の代わりに，次のような外観にすることができる。

http://yoursite/staticpages/index.php/20030313100131123

検索エンジンの中には，URLに '?' や '=' などの文字が含まれると，そのページをインデックスに登録しないものがあるということが知られている。これをさらに推し進めて，数字で構成されたページIDを（できれば，ページの内容と関連する）単語や表現に置き換えることで，検索エンジンにさらに登録されやすくすることもできるだろう。

http://yoursite/staticpages/index.php/about

URL書き換えを行うには，「管理者用メニュー - コンフィギュレーション - Geeklog - サイト」で「URLリライト」を「はい」にする必要がある。

所収者の削除と共に削除する

Geeklogのすべてのオブジェクト同様，静的ページには所有者（そのページの作成者）がいる。そのユーザのアカウントが何らかの理由で削除された場合，そのユーザが所有していた静的ページも削除するか，GeeklogのRootグループに属する人に引き継がせることができる。

「管理者用メニュー - コンフィギュレーション - 静的ページ」で「所有者の削除と共に削除する」を「いいえ」（初期値）にすると，所有者が削除されても作成した静的ページは削除されず，Rootグループのメンバ（ユーザIDが最も若い人，つまり，たいていはサイトの管理者）に引き継がれる。「はい」にすると。所有者のアカウントが削除されたとき，作成した静的ページも削除される。
